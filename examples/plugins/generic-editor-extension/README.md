# Generic Editor Extension Plugin Example

Starter template for external YANTA plugins.
Use this folder as your scaffold for new plugins.

For full plugin policy and workflow (build, install, ship), use:

- `examples/plugins/plugin-development.md`

## What This Example Shows

- command contribution
- TipTap extension contribution
- BlockNote extension contribution
- slash menu contribution
- lifecycle hooks

## Files

- `plugin.ts` - plugin code
- `plugin.toml` - manifest
- `package.json` - build config for this plugin

## Manifest

Current values:

- `id = "example.generic-editor-extension"`
- `api_version = "1"`
- `entry = "main.js"`

## Build

From this folder:

```bash
npm install
npm run build
```

Output:

- `main.js`
- `main.meta.json`

## Validate

1. Open command palette and run `Plugin: Generic Editor Extension`.
2. In editor type `/` and run `Insert Example Marker`.

## Adapting for Real Plugins

1. Copy this folder into your own repo/plugin directory.
2. Replace sample extension code with your real extension package code.
3. Keep plugin entry output as `main.js`.
4. Keep `main.meta.json` generated by `yanta-plugin build`.
5. Keep `setup(api)` export contract.
6. Verify compatibility with YANTA editor runtime before release.
7. If moved outside this repo, switch build script from `go run .../cmd/yanta-plugin` to `yanta-plugin build ...`.
